# 技術調査

ユーザがライブラリ、フレームワーク、SDK、API、CLI ツール、またはクラウドサービス(React、Next.js、Prisma、Express、Tailwind、Django、Spring Boot などの有名なものを含む)について質問した際は、常に `ctx7` CLI を使用して最新のドキュメントを取得すること。

これには、API 構文、設定方法、バージョンの移行、ライブラリ固有のデバッグ、セットアップ手順、CLI ツールの使用方法などが含まれる。たとえ答えを知っていると思われる場合でも、学習データには最新の変更が反映されていない可能性があるため、この手順を優先すること。

ライブラリのドキュメントに関しては、Web 検索よりもこちらを優先して使用すること。ただし、リファクタリング、ゼロからのスクリプト作成、ビジネスロジックのデバッグ、コードレビュー、一般的なプログラミング概念の質問には使用しないこと。

### 実行手順

1. ライブラリの特定の為に `deno run --allow-env --allow-sys --allow-read --allow-net=context7.com npm:ctx7@latest library <name> "<user's question>"` を実行する。
2. 最適なライブラリ ID (形式: `/org/project`)を選択する。
3. ドキュメントを取得する為に `deno run --allow-env --allow-sys --allow-read --allow-net=context7.com npm:ctx7@latest docs <libraryId> "<user's question>"` を実行する。
4. 取得したドキュメントに基づいて回答する。

### 注意事項

- MUST: **クエリに機密情報(API キー、パスワード、認証情報)を含めないこと**。
- ユーザから `/org/project` 形式で直接 ID が提供されない限り、必ず最初に library コマンドを実行して有効な ID を取得すること。
- 質問内容は、「nextjs」ではなく「Next.js」、「customerio」ではなく「Customer.io」、「threejs」ではなく「Three.js」など、正しい句読点を用いた公式のライブラリ名を使用すること。
- ライブラリの選択基準は、正式名称の一致、説明の関連性、コードスニペットの数、ソースの信頼性(High / Medium を推奨)、ベンチマークスコア(高いほど良い)で決定する。
- クエリにはユーザの質問全文を使用すること。具体的かつ詳細なクエリの方が、曖昧な単語よりも良い結果が得られる。
- 1 つの質問につき 3 回を超えてコマンドを実行しないこと。
- バージョン固有のドキュメントが必要な場合は、library の出力結果にある `/org/project/version` 形式(例: `/vercel/next.js/v14.3.0`)を使用すること。
- コマンドがクォータエラーで失敗した場合は、ユーザーにその旨を伝え、制限を引き上げるよう提案すること。勝手にトレーニングデータの内容で補完して回答を返すようなことはしないこと。
